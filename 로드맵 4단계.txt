1. 기획 및 데이터 구조화 (설계 단계)
코드를 작성하기 전, AI에게 명확한 지시를 내리기 위해 무엇을 만들지 정의해야 합니다.

기능 명세: 장비 관리(서버, 스위치 등), 문서 관리(PDF, HWP 등), 점검 보고서 생성, 업무 절차서(Wiki 형태) 등 필요한 기능을 리스트업 합니다.

데이터베이스 설계: '서버'라는 자산이 가질 속성(IP, OS, 위치, 구매일 등)과 각 데이터 간의 관계를 정의합니다.

2. 오프라인 호환 기술 스택 선정 (환경 설정)
인터넷이 되지 않는 환경에서는 외부 라이브러리나 CDN(Content Delivery Network)을 사용할 수 없습니다. 따라서 모든 구성 요소가 내장된(Self-hosted) 기술을 선택해야 합니다.

Backend: Python (Django/FastAPI) 또는 Node.js 등 설치 파일을 쉽게 구할 수 있는 언어.

Frontend: 외부 인터넷 연결 없이 동작하는 React나 Vue.js (빌드 파일 형태).

Database: 설치형 DB (PostgreSQL, MariaDB) 또는 파일 기반 DB (SQLite - 소규모일 경우).

Container: Docker 사용을 강력 추천합니다. 외부에서 개발한 환경을 그대로 '이미지'로 만들어 오프라인 망으로 옮기기에 가장 적합합니다.

3. 바이브 코딩을 통한 개발 (구현 단계)
AI(ChatGPT, Claude, Cursor 등)를 활용하여 실제 코드를 작성합니다. 이때 중요한 것은 보안과 모듈화입니다.

프롬프트 전략: "발전소 유지보수 시스템을 만들 건데, Django 모델로 '서버' 자산 관리 테이블을 짜줘"와 같이 구체적으로 요청합니다.

오프라인 대응: AI에게 코드를 요청할 때 항상 "CDN 링크 대신 로컬 정적 파일(Static file)을 사용하는 방식으로 짜줘"라고 명시해야 합니다.

4. 이관 및 배포 (오프라인 배치)
외부 인터넷 망에서 개발한 결과물을 발전소 내부망으로 옮기는 절차입니다.

패키징: 개발된 소스 코드와 필요한 모든 라이브러리(Dependencies)를 하나의 Docker 이미지나 압축 파일로 만듭니다.

반입 절차: 보안 규정에 따라 USB나 망간 자료 전송 시스템을 통해 파일을 내부로 반입합니다.

구동: 내부 서버에 파일을 올리고 시스템을 실행합니다.

이 프로젝트를 성공적으로 시작하기 위해, 가장 먼저 논의하고 싶은 부분을 선택해 주세요.

기능 및 DB 상세 설계: "서버랑 스위치 관리할 때 어떤 항목들이 DB에 들어가야 할지 같이 고민해 줘."

오프라인 기술 스택 확정: "Docker랑 파이썬을 쓴다면 구체적으로 어떻게 세팅해야 인터넷 없이 돌아가는지 알려줘."

바이브 코딩 프롬프트 전략: "AI한테 뭐라고 질문해야 내가 원하는 유지보수 웹 화면을 바로 만들어주는지 예시를 보여줘."
